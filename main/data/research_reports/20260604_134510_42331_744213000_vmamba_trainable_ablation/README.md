## Pipeline

The neural input is built before the model: CanonSort orders the graph context, then tensorization creates C0-C1 for VMamba-T or C0-C5 for VMamba-Mesh-T. The tensor then enters patch embedding and VSS-style SS2D scan blocks. The pair head combines the learned embeddings with an auditable structural calibration vector, emits a sigmoid score, and applies the calibrated threshold to produce `predict_pairs(graph)`.

Hard-negative mining in this run used the auditable `structural_plus_llm_manifest` strategy. Codex/GPT-5 acted as the LLM-assisted hard-negative reviewer and produced the manifest `research/vmamba/manifests/llm_hard_negatives_article_iv_20260604.json`; the training process did not call an LLM at runtime. Instead, it loaded that JSON, prioritized the suggested non-duplicate pairs, recorded the agent/model/manifest id in each artifact, and fell back to the structural miner only when more negatives were needed to satisfy `negative_ratio=16`.

The LLM suggestion criteria were: same SOR/SOT/SPEC layer, same domain suffix when available, similar naming tokens, similar local degree/context size and high VMamba-Mesh structural feature score. Example selected pairs include `SPEC_store_sales_summary_D2` vs. `SPEC_web_sales_summary_D2`, which is a hard negative because it looks semantically and structurally close but is not labeled as a duplicate.
